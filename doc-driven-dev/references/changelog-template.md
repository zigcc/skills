# CHANGELOG Update Template

## Session Entry Format

```markdown
### Session YYYY-MM-DD-NNN

**Date**: YYYY-MM-DD
**Goal**: Brief description

#### Completed Work
1. Item 1
2. Item 2

#### Test Results
- Unit tests: X tests passed
- Integration tests: passed/failed

#### Next Steps
- [ ] Task 1
- [ ] Task 2
```

## Version Release Format

```markdown
## [vX.Y.Z] - YYYY-MM-DD

### Added
- New feature 1
- New feature 2

### Changed
- Modified behavior 1

### Fixed
- Bug fix 1

### Deprecated
- Deprecated feature 1

### Removed
- Removed feature 1

### Security
- Security fix 1
```

## Guidelines

1. **Session entries**: Use for daily development tracking
2. **Version entries**: Use for formal releases
3. **Be specific**: Include test counts, file names, function names
4. **Link to Stories**: Reference the corresponding Story file
5. **Keep chronological**: Newest entries at top
